🔄 Case Routing Examples

Automatic routing based on workflow rules

Example 1: Route to Specific User

Route a case to a specific user by user ID.

// Route case to user ID 5 fetch('/public/api/case_routing.php?action=route_to_user', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ case_ref: 'EDU-ADM-20260324-ABC123', user_id: 5, reason: 'Assigned for review' }) }) .then(res => res.json()) .then(data => console.log(data));

Example 2: Route to Group

Route a case to a user group (e.g., reviewers, admins).

// Route case to reviewers group fetch('/public/api/case_routing.php?action=route_to_group', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ case_ref: 'EDU-ADM-20260324-ABC123', group_name: 'portal_reviewer', reason: 'Requires expert review' }) }) .then(res => res.json()) .then(data => console.log(data));

Example 3: Auto-Route with Conditional Rules

Automatically route based on form data conditions.

// Auto-route based on program selection const rules = [ { condition: { field: 'program', operator: '==', value: 'Engineering' }, route_to: 'engineering_reviewers', route_type: 'group', reason: 'Engineering program application' }, { condition: { field: 'gpa', operator: '>=', value: 3.5 }, route_to: 'honors_committee', route_type: 'group', reason: 'High GPA - Honors consideration' } ]; fetch('/public/api/case_routing.php?action=auto_route', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ case_ref: 'EDU-ADM-20260324-ABC123', rules: rules }) }) .then(res => res.json()) .then(data => console.log(data));

Example 4: Integration with DX Interpreter

Automatically route after form completion.

const app = new DXInterpreter('#container', { dx_id: 'admission_case', renderMode: 'modal', onComplete: async function(data) { // Auto-route based on submitted data const rules = [ { condition: { field: 'program', operator: '==', value: 'Engineering' }, route_to: 'engineering_reviewers', route_type: 'group', reason: 'Engineering application' }, { condition: { field: 'international_student', operator: '==', value: 'yes' }, route_to: 'international_office', route_type: 'group', reason: 'International student' } ]; const response = await fetch('/public/api/case_routing.php?action=auto_route', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ case_ref: data.case_ref, rules: rules }) }); const result = await response.json(); if (result.status === 'success') { alert('Application submitted and routed to ' + result.data.assigned_to_group); } } });

💡 Tip: This pattern allows you to automatically route cases to the appropriate team based on the applicant's responses, without any manual intervention.

📚 Routing Operators Reference

Operator Description Example
== Equals { field: 'status', operator: '==', value: 'approved' }
!= Not equals { field: 'status', operator: '!=', value: 'rejected' }
> Greater than { field: 'gpa', operator: '>', value: 3.0 }
< Less than { field: 'age', operator: '<', value: 18 }
>= Greater than or equal { field: 'score', operator: '>=', value: 80 }
<= Less than or equal { field: 'attempts', operator: '<=', value: 3 }
contains String contains { field: 'email', operator: 'contains', value: '@university.edu' }
in Value in array { field: 'program', operator: 'in', value: ['CS', 'IT', 'SE'] }
